Command grammar (refined)

Your draft is close. Three refinements make it unambiguous and consistent:

  • Imperative verbs, not past tense: start / stop / done — not started / completed. Reads as a command, scripts cleanly.
  • Separate "run the app" from "start work": devteam run boots the app locally; devteam ticket <id> start begins a work session. Otherwise "start" is overloaded.
  • Noun → id → verb stays your shape (devteam ticket 123 push), and bare devteam launches the TUI.

devteam — your DevTeam hub, from the terminal.

USAGE
  devteam [command]
  devteam                       # no command → launch the interactive TUI (chat)

WORK
  chat          Launch the interactive TUI            (default)
  ticket        Tickets + their git branch            open·start·stop·done·commit·push·pr
  document      Create & manipulate documents         create·translate·rewrite·ls
  run           Start the app locally                 run·run stop·run logs
  test          Run tests for a file or folder
  sync          Pull + rebase the current branch
  commit        Stage & commit (AI message if no -m)
  push          Push current branch to origin
  pr            Open / update the pull request

WORKSPACE
  init          Link this folder to a DevTeam project
  projects      List & switch projects
  status        Folder · branch · ticket · PR at a glance
  config        Get/set config (server, token, editor…)
  login         Authenticate to your DevTeam server

GLOBAL FLAGS
  --project <slug>   Override the active project
  --folder <path>    Project folder            (default: .)
  --json             Machine-readable output
  -y, --yes          Skip confirmation prompts
  --no-color         Disable ANSI color
  -v, --verbose      Verbose logging
  -h, --help         Help for any command
  --version          Print version

EXAMPLES
  $ devteam ticket 123 open          # checkout the ticket's branch
  $ devteam ticket 123 start         # status → in-progress, start timer
  $ devteam ticket 123 commit        # AI commit message
  $ devteam ticket 123 push          # push + refresh PR
  $ devteam test ./app/models        # run a folder's tests
  $ devteam run                      # boot the app locally
  $ devteam document 88 translate --to he

Run "devteam <command> --help" for details · run "devteam" for the TUI.

devteam ticket --help
$ devteam ticket --help

Work with a ticket and the git branch behind it.

USAGE
  devteam ticket <id> <action> [flags]
  devteam ticket new  [--title …] [--kind story|bug]
  devteam ticket ls   [--mine] [--sprint current]

ACTIONS
  show            Status, assignee, branch, PR, checks
  open            Switch to the ticket branch (create from --base if nil)
  start           Status → in-progress · start work session
  stop            Pause the work session
  done            Status → done            (alias: complete)
  status <value>   backlog|open|in_progress|in_review|testing|done|blocked
  commit [-m]      Commit staged changes   (AI message if -m omitted)
  push            Push branch + create/refresh PR
  pr [--draft]     Open a pull request
  watch           Stream CI + review status

FLAGS
  --base <branch>   Base for new branches   (default: main)
  -m, --message     Commit / PR message
  --no-switch       Don't change the local branch on open

EXAMPLES
  $ devteam ticket 123 open
  $ devteam ticket 123 commit -m "Fix null queue race"
  $ devteam ticket 123 push
Branch behavior on open. If the ticket already has branch_name, devteam checks it out (fetching if needed). If it's nil, devteam creates feature/t-<id>-<slug> from --base, checks it out, and writes the branch back to the ticket via the API — so the next teammate who runs open lands on the same branch.

Switching project re-reads the folder & git state

Selecting a project in the left rail (or ^P) points devteam at that project's local folder and refreshes the context panel: current branch + ahead/behind + dirty flag, the active ticket, and the open PR (with CI/review checks). If the folder isn't linked yet, the panel shows not linked — run `devteam init` with a one-key fix.
