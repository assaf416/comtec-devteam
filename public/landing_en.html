Testing Integration

BDD with Cucumber — baked in, not bolted on

Cucumber feature files live alongside your code. CI parses results directly into DevTeam Hub. The AI reviews your Gherkin and suggests missing scenarios before you push.

Your .feature files drive the board

🔌 Full REST API

Every feature is reachable from CI, scripts, the CLI, or your own integrations. Bearer token auth — one endpoint for everything.

GET/api/v1/tickets

List tickets. Filter by assignee, status, project, sprint.

POST/api/v1/tickets

Create a ticket with full metadata — project, priority, kind, assignee.

PATCH/api/v1/tickets/:id

Update status, assignee, branch name, or any ticket field.

GET/api/v1/projects

List projects with repo URL, default branch, and sprint info.

GET/api/v1/sprints/:id

Sprint data — tickets, points, burn-down, velocity, CI pass rate.

POST/tools/ai/code_review

Submit a diff for AI code review. Returns verdict, score, Markdown findings.

POST/tools/ai/test_review

Submit a .feature file for AI Cucumber review and missing-scenario suggestions.

GET/api/v1/token

Get or regenerate the current user's API token.

# Create a ticket from CI or an incident script curl -s -X POST "$DEVTEAM_URL/api/v1/tickets" \ -H "Authorization: Bearer $TOKEN" \ -H "Content-Type: application/json" \ -d '{ "ticket": { "project_id": 3, "title": "DB timeout on checkout", "status": "open", "priority": "high", "kind": "bug_fix" } }' # AI code review from a CI pipeline step curl -s -X POST "$DEVTEAM_URL/tools/ai/code_review" \ -H "Authorization: Bearer $TOKEN" \ --data-urlencode "language=go" \ --data-urlencode "diff@-" < my.diff
Developer CLI

devteam — the whole platform from your terminal

A single Go binary. No runtime, no Node, no Python. Works on Mac, Linux, and Windows. Now with Cucumber tests via godog.

$ devteam ticket 42 open ✓ T-42 "Fix login bug" ✓ branch feature/t-42-fix-login-bug created from main and checked out ↳ `devteam ticket 42 start` to begin a work session $ devteam ticket 42 start ✓ work session started — T-42 is now in_progress $ devteam ticket 42 commit -m "handle nil user on checkout" ✓ committed: T-42: handle nil user on checkout $ devteam ticket 42 push ✓ pushed → origin/feature/t-42-fix-login-bug $ devteam ticket 42 done ✓ marked done — T-42 is now done $ devteam status folder /home/dev/projects/api-core branch feature/t-42-fix-login-bug ↑1 ↓0 clean $ devteam test ./app/models • running: bundle exec rspec app/models ✓ tests passed
🎫

ticket

Open, start, stop, commit, push, done — full ticket lifecycle from the shell.

🧪

test

Run tests for a file or folder. Configured per repo via .devteam.yml.

📦

commit / push

Stage, commit with a ticket-prefixed message, and push — in three short commands.
