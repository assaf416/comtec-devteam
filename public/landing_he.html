סוף ספרינט
ניהול טיקטים

כל יחידת עבודה — עם מעקב מלא

טיקטים עוברים מחזור חיים ברור — מטיוטה דרך אישור, הערכה, פיתוח ובדיקות — עד "בוצע". שום דבר לא נופל בין הכיסאות.

  • מחזור חיים מלא: טיוטה ← אושר ← הוערך ← בפיתוח ← בוצע
  • עדיפות, סוג, מורכבות, שלוח ובעלים בכל טיקט
  • Git branch נוצר אוטומטית מכותרת הטיקט
  • קישור ל-CI runs, PRs, פריסות ומשימות
  • בדיקת מוכנות AI — מחזיר טיקטים מעורפלים לבעלים
  • פירוק משימות עם הערכות זמן מכוילות
ניהול ספרינטים

איטרציות צפויות, התקדמות גלויה

ספרינט אחד פעיל בכל פעם. תרשימי Burn-down, מעקב ווולוסיטי וציון בריאות AI מספרים לכם אם הספרינט על המסלול — לפני שמאוחר לפעול.

  • ספרינט נוכחי יחיד עם יעדים ברורים ו-Burn-down
  • דוחות ווולוסיטי על פני ספרינטים — זיהוי מגמות מוקדם
  • ניתוח ספרינט AI נטען חי בדף הספרינט
  • ניקוד דיוק הערכות לפי מפתח ולפי רמת מורכבות
  • לוח שנה של ספרינט עם תזמון אוטומטי של טקסים
אינטגרציית בדיקות

BDD עם Cucumber — מובנה, לא מחובר מבחוץ

קבצי Feature של Cucumber חיים לצד הקוד שלכם. CI מפרסר תוצאות ישירות ל-DevTeam Hub. ה-AI בודק את ה-Gherkin שלכם ומציע תרחישים חסרים לפני שאתם מעלים.

קבצי ה-.feature שלכם מניעים את הלוח

Feature: Ticket commands # As a developer # I want to manage tickets from the CLI Background: Given the API server is running And I am authenticated Scenario: Start work on a ticket Given a ticket with id 42, status "open" When I run "devteam ticket 42 start" Then the output contains "in_progress" And the exit code is 0 Scenario: Mark a ticket as done Given a ticket with id 42, status "in_progress" When I run "devteam ticket 42 done" Then the output contains "done" And the exit code is 0

תוצאות CI מפורסרות לתוך הפלטפורמה

כל CI run מפרסם תוצאות חזרה ל-DevTeam Hub. מגמות כיסוי קוד, שיעורי כישלון וזיהוי בדיקות flaky גלויים בדשבורד CI — מקושרים לטיקט שגרם לכישלון.

בדיקת מוכנות טיקט

שופט את הטיקט מול Definition of Ready — סיפור ברור, קריטריוני קבלה, שלבי שחזור לבאגים. מחזיר טיקטים מעורפלים אוטומטית לבעלים.

POST /tools/ai/ticket_quality

🔍

Code Review (Go / Ruby / C# / Node)

בוחן git diff לאיתור באגים, אבטחה, ביצועים ואידיומים של השפה. מקבץ ממצאים ל-Blocking, Suggestions ו-Lint.

POST /tools/ai/code_review

🧪

סקירת בדיקות Cucumber

בוחן Gherkin לבהירות, דטרמיניזם ומבנה. מציע תרחישי edge-case חסרים — כתובים כ-Gherkin מוכן להדבקה.

POST /tools/ai/test_review

💡

הצעת פתרון

קורא טיקט ומציע גישת מימוש — צעדים, קבצים סבירים, edge cases, סיכונים ואיזה בדיקות להוסיף.

POST /tools/ai/solution_suggestion

🐛

תיקון הבאג

קורא טיקט באג, מנתח את הסיבה השורשית הסבירה ומציע תיקון מינימלי קונקרטי עם בדיקות. ציון ביטחון 0–100.

POST /tools/ai/fix_bug

יצירת משימות והערכות

מפרק סיפור למשימות Task עצמאיות הניתנות להערכה. מכייל הערכות מול נתוני הדיוק ההיסטוריים של הצוות.

POST /tools/ai/generate_tasks

ניתוח בריאות ספרינט

🔌 REST API מלא

כל פיצ'ר נגיש מ-CI, סקריפטים, ה-CLI או אינטגרציות משלכם. אותנטיקציית Bearer token — נקודת קצה אחת לכל הדברים.

GET/api/v1/tickets

רשימת טיקטים. סינון לפי שלוח, סטטוס, פרויקט, ספרינט.

POST/api/v1/tickets

יצירת טיקט עם מטאדאטה מלאה — פרויקט, עדיפות, סוג, שלוח.

PATCH/api/v1/tickets/:id

עדכון סטטוס, שלוח, שם branch או כל שדה בטיקט.

GET/api/v1/projects

רשימת פרויקטים עם repo URL, branch ברירת מחדל ומידע ספרינט.

GET/api/v1/sprints/:id

נתוני ספרינט — טיקטים, נקודות, burn-down, ווולוסיטי, שיעור CI.

POST/tools/ai/code_review

שליחת diff לסקירת קוד AI. מחזיר verdict, ציון וממצאי Markdown.

POST/tools/ai/test_review

שליחת קובץ .feature לסקירת Cucumber AI והצעת תרחישים חסרים.

GET/api/v1/token

קבלה או חידוש של API token של המשתמש הנוכחי.

# יצירת טיקט מ-CI או סקריפט אינציידנט curl -s -X POST "$DEVTEAM_URL/api/v1/tickets" \ -H "Authorization: Bearer $TOKEN" \ -H "Content-Type: application/json" \ -d '{ "ticket": { "project_id": 3, "title": "DB timeout בצ\'אקאוט", "priority": "high" } }' # סקירת קוד AI משלב ב-CI pipeline curl -s -X POST "$DEVTEAM_URL/tools/ai/code_review" \ -H "Authorization: Bearer $TOKEN" \ --data-urlencode "language=go" \ --data-urlencode "diff@-" < my.diff
CLI למפתחים

devteam — כל הפלטפורמה מהטרמינל

קובץ בינארי Go יחיד. ללא runtime, ללא Node, ללא Python. עובד על Mac, Linux ו-Windows. כעת עם בדיקות Cucumber דרך godog.

$ devteam ticket 42 open ✓ T-42 "תיקון באג לוגין" ✓ branch feature/t-42-fix-login-bug נוצר מ-main ו-checkout בוצע ↳ `devteam ticket 42 start` להתחלת ה-work session $ devteam ticket 42 start ✓ work session התחיל — T-42 עכשיו in_progress $ devteam ticket 42 commit -m "handle nil user on checkout" ✓ committed: T-42: handle nil user on checkout $ devteam ticket 42 push ✓ pushed → origin/feature/t-42-fix-login-bug $ devteam ticket 42 done ✓ מסומן כ-done — T-42 עכשיו done $ devteam status folder /home/dev/projects/api-core branch feature/t-42-fix-login-bug ↑1 ↓0 clean $ devteam test ./app/models • מריץ: bundle exec rspec app/models ✓ בדיקות עברו

ticket
