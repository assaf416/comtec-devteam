1. Current Logging Issues
Each application logs differently — no unified format, no central storage, no cross-app correlation.

🔴 Fragmented Formats

Rails logs plain text, .NET uses structured EventLog, NestJS uses Winston JSON, Next.js uses console.log. No common schema — impossible to search across apps.

🔴 No Central Storage

Logs live on each server's filesystem. SSH required to read them. Log rotation deletes old entries. No searchable archive.

🔴 No Correlation

No request IDs, trace IDs, or correlation tokens flow between services. Debugging cross-app issues requires manual timestamp matching.

2. Requirements
What the unified logging solution must provide.

Must Have

  • On-Premises — no log data leaves the network
  • Unified JSON format — same schema for all 5 apps
  • Central search & query — search all logs from one UI
  • Correlation IDs — trace requests across services
  • CI integration — Jenkins build logs collected too
  • Retention policy — configurable per environment
  • Alerting — notify on error spikes
  • Zero / low cost — open-source preferred

Nice to Have

  • Dashboard with charts & metrics
  • Log-based metrics (error rates, latency percentiles)
  • Live tail (real-time streaming)
  • API for DevTeam Hub integration
  • Windows agent for .NET 4.8.1 apps
  • Multi-tenant label isolation per project

Standard Log Schema

4. Log Management Tool Landscape
Evaluated tools for centralized log aggregation, search, and visualization.

Grafana Loki Recommended

Log aggregation system by Grafana Labs. Does NOT index log content — indexes only labels (service, env, level). Extremely storage-efficient. Native Grafana integration.

Free & OSS On-Prem Docker LogQL

Elasticsearch + Kibana (ELK)

Full-text indexing of all log fields. Extremely powerful search but heavy on resources (RAM, disk). Elastic license changed — OSS fork is OpenSearch.

Heavy Resources Full-Text Search

Graylog

Centralized log management with GELF protocol, dashboards, and alerts. Requires Elasticsearch + MongoDB. More complex to deploy.

3 Services Required GELF Protocol

Fluentd / Fluent Bit

Log collectors / forwarders. Not storage — they ship logs to a backend (Loki, Elasticsearch, etc.). Fluent Bit is lightweight C agent. Alternative to Promtail.

Collector Only CNCF Project

Seq (Datalust)

Structured log server optimized for .NET (Serilog). Single-user license free. Excellent for .NET-only shops but limited multi-language support.

.NET Focused Free Single User

Datadog / Splunk (Cloud)

Enterprise cloud log platforms. Powerful but expensive and send data off-premises. Not suitable for our on-prem requirement.

Cloud Only Expensive

8. Rails Configuration — Structured JSON Logging
Add the lograge gem and configure JSON output with correlation IDs.

Gemfile

# Structured JSON logging
gem "lograge"
gem "lograge-sql"    # optional: SQL query logging

config/initializers/lograge.rb

Rails.application.configure do
  config.lograge.enabled = true
  config.lograge.formatter =
    Lograge::Formatters::Json.new

  config.lograge.custom_options = lambda do |event|
    {
      timestamp:      Time.current.iso8601(3),
      service:        "devteam-hub",
      environment:    Rails.env,
      host:           Socket.gethostname,
      version:        ENV.fetch("APP_VERSION",
                                "dev"),
      correlation_id: event.payload[:correlation_id],
      request_id:     event.payload[:request_id],
      user_id:        event.payload[:user_id],
      ip:             event.payload[:ip],
      user_agent:     event.payload[:user_agent]
    }
  end

  config.lograge.custom_payload do |controller|
    {
      correlation_id: controller
        .request.headers["X-Correlation-ID"] ||
        controller.request.request_id,
      request_id: controller.request.request_id,
      user_id: controller.try(:current_user)&.id,
      ip: controller.request.remote_ip,
      user_agent: controller.request.user_agent
    }
  end
end

def set_correlation_id

Rails Controller — Api::V1::LogsController

# app/controllers/api/v1/logs_controller.rb
module Api
  module V1
    class LogsController < BaseController
      LOKI_URL = ENV.fetch("LOKI_URL", "http://loki:3100")

      # GET /api/v1/logs
      def index
        query = build_logql_query(params)
        response = loki_query(query,
          start_time: params[:from],
          end_time:   params[:to],
          limit:      params.fetch(:limit, 100).to_i
        )
        render json: { logs: response[:entries], total: response[:total], has_more: response[:has_more] }
      end

      # GET /api/v1/logs/services
      def services
        labels = loki_labels("service")
        render json: { services: labels }
      end

      # POST /api/v1/logs/push
      def push
        entries = params.require(:entries)
        accepted = push_to_loki(entries)
        render json: { accepted: accepted, rejected: entries.size - accepted }
      end

      # GET /api/v1/logs/stats
      def stats
        render json: {
          error_count_24h:  loki_count('{level="error"}', "24h"),
          warn_count_24h:   loki_count('{level="warn"}',  "24h"),
          volume_by_service: loki_volume_by_label("service", "24h"),
          top_errors:        loki_top_messages('{level="error"}', "24h", 10)
        }
      end

      private

      def build_logql_query(params)
        labels = []
        labels << %Q(service="#{params[:service]}") if params[:service].present?
        labels << %Q(level="#{params[:level]}")      if params[:level].present?
        selector = "{#{labels.join(',')}}"
        params[:query].present? ? "#{selector} |= `#{params[:query]}`" : selector
      end

      def loki_query(query, start_time:, end_time:, limit:)
        uri = URI("#{LOKI_URL}/loki/api/v1/query_range")
        uri.query = URI.encode_www_form(
          query: query, start: start_time, end: end_time, limit: limit
        )
        resp = Net::HTTP.get_response(uri)
        parse_loki_response(resp)
      end

      def push_to_loki(entries)
        streams = entries.group_by { |e| e[:service] }.map do |svc, logs|
          {
            stream: { service: svc, environment: logs.first[:environment], level: logs.first[:level] },
            values: logs.map { |l| [(Time.parse(l[:timestamp]).to_f * 1e9).to_i.to_s, l.to_json] }
          }
        end
        uri = URI("#{LOKI_URL}/loki/api/v1/push")
        req = Net::HTTP::Post.new(uri, "Content-Type" => "application/json")
        req.body = { streams: streams }.to_json
        Net::HTTP.start(uri.hostname, uri.port) { |http| http.request(req) }
        entries.size
      end
    end
  end
end

class Logs < Thor
  desc "list", "Query application logs"
  option :service,  type: :string,  desc: "Filter by service name"
  option :level,    type: :string,  desc: "Filter by log level"
  option :from,     type: :string,  desc: "Start time (ISO 8601 or relative)"
  option :to,       type: :string,  desc: "End time"
  option :query,    type: :string,  desc: "Text search in log messages"
  option :limit,    type: :numeric, default: 50
  option :follow,   type: :boolean, default: false, desc: "Live tail"
  option :json,     type: :boolean, default: false, desc: "Raw JSON output"
  def list
    params = options.slice(:service, :level, :from, :to, :query, :limit)
    resp = DevTeam.api(:get, "/logs", params: params)
    if options[:json]
      puts JSON.pretty_generate(resp["logs"])
    else
      resp["logs"].each { |l| print_log_line(l) }
    end
  end
  default_task :list

  desc "services", "List available log services"
  def services
    resp = DevTeam.api(:get, "/logs/services")
    resp["services"].each { |s| puts "  #{s}" }
  end

  desc "stats", "Show log statistics (last 24h)"
  def stats
    resp = DevTeam.api(:get, "/logs/stats")
    puts "Errors (24h):   #{resp['error_count_24h']}"
    puts "Warnings (24h): #{resp['warn_count_24h']}"
    puts "\nVolume by service:"
    resp["volume_by_service"]&.each { |s, c| puts "  #{s}: #{c}" }
  end

  desc "push", "Push a log entry"
  option :service,  type: :string,  required: true
  option :level,    type: :string,  default: "info"
  option :message,  type: :string,  required: true
  def push
    DevTeam.api(:post, "/logs/push", body: {
      entries: [{
        timestamp: Time.now.utc.iso8601,
        service:   options[:service],
        level:     options[:level],
        message:   options[:message],
        environment: "production"
      }]
    })
    puts "✓ Log pushed"
  end
end
11. CI Integration — Jenkins & GitHub Actions
How to connect CI build logs to the unified logging system.

Jenkins Pipeline — Loki Log Plugin

// Jenkinsfile — post-build log push
pipeline {
  agent any
  environment {
    LOKI_URL = "http://loki:3100"
  }

  stages {
    stage('Build') {
      steps {
        sh 'dotnet build'
      }
    }
    stage('Test') {
      steps {
        sh 'dotnet test --logger "trx"'
      }
    }
  }

  post {
    always {
      // Push build log to Loki
      script {
        def logText = currentBuild
          .rawBuild.getLog(1000).join("\n")
        def payload = [
          streams: [[
            stream: [
              service: "jenkins",
              job: env.JOB_NAME,
              build: env.BUILD_NUMBER,
              status: currentBuild.result
            ],
            values: [
              ["${System.nanoTime()}", logText]
            ]
          ]]
        ]
        httpRequest(
          url: "${LOKI_URL}/loki/api/v1/push",
          httpMode: 'POST',
          contentType: 'APPLICATION_JSON',
          requestBody:
            groovy.json.JsonOutput
              .toJson(payload)
        )
      }

      // Notify DevTeam Hub
      sh """
        curl -X POST \\
          ${DEVTEAM_URL}/api/v1/logs/push \\
          -H "Authorization: Bearer \${TOKEN}" \\
          -H "Content-Type: application/json" \\
          -d '{"entries":[{
            "service":"jenkins",
            "level":"info",
            "message":"Build #${BUILD_NUMBER} ${currentBuild.result}"
          }]}'
      """
    }
  }
}

Next.js 16 (Work Mgmt System) — Pino

// lib/logger.ts
import pino from 'pino';

export const logger = pino({
  level: process.env.LOG_LEVEL || 'info',
  base: {
    service: 'work-mgmt',
    environment: process.env.NODE_ENV,
    version: process.env.APP_VERSION,
  },
  timestamp: pino.stdTimeFunctions.isoTime,
  formatters: {
    level(label) {
      return { level: label };
    },
  },
  redact: ['req.headers.authorization'],
});

// Middleware for correlation ID
export function correlationMiddleware(
  req, res, next
) {
  req.correlationId =
    req.headers['x-correlation-id'] ||
    crypto.randomUUID();
  res.setHeader(
    'X-Correlation-ID', req.correlationId
  );
  next();
}
