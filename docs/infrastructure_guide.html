Chapter 1
Git — Gitea
🌿
Gitea
Self-hosted Git Service

A lightweight, self-hosted Git service written in Go. Provides repositories, pull requests, issues, webhooks, and a REST API — all running on-prem with minimal resources.

🔗
GiteaService
Rails Integration Class

A Faraday-based service class in app/services/gitea_service.rb that wraps every Gitea API call — branch creation, PR data, diff fetching, webhook management.

What Gitea Does

Gitea acts as the central source control server for all projects. It stores code repositories, accepts pushes from developers, fires webhook events to the Rails app (push, pull_request, release), and exposes a full REST API for programmatic access.

How It Connects to Rails

Webhook Registration
When a project is linked to a Gitea repo, the app calls GiteaService#create_webhook to register a webhook pointing at /webhooks/gitea.
Incoming Webhooks
WebhooksController#gitea receives push/PR events, verifies the HMAC-SHA256 signature, and updates tickets, branches, and pull requests accordingly.
Outgoing API calls
CreateBranchJob and SyncPullRequestJob call GiteaService to create branches and pull full PR diff/file/comment data.

Configuration

bash (.env)# .env / credentials / ENV
GITEA_URL=http://gitea.local:3001
GITEA_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxx  # user access token
GITEA_WEBHOOK_SECRET=a_random_secret_string

GiteaService — Full Implementation

rubyclass GiteaService
  BASE_URL  = ENV.fetch("GITEA_URL",   "http://localhost:3001")
  API_TOKEN = ENV.fetch("GITEA_TOKEN", "")

  def initialize
    @conn = Faraday.new(url: BASE_URL) do |f|
      f.request  :json
      f.response :json
      f.headers["Authorization"] = "token #{API_TOKEN}"
    end
  end

  # Create a branch for a ticket
  def create_branch(repo_owner:, repo_name:, branch_name:, base_branch: "main")
    @conn.post("/api/v1/repos/#{repo_owner}/#{repo_name}/branches") do |req|
      req.body = { new_branch_name: branch_name, old_branch_name: base_branch }
    end.then { |r| r.success? ? r.body : nil }
  end

  # Fetch PR diff (unified format)
  def pull_request_diff(repo_owner:, repo_name:, pr_number:)
    Faraday.get(
      "#{BASE_URL}/api/v1/repos/#{repo_owner}/#{repo_name}/pulls/#{pr_number}.diff",
      {}, { "Authorization" => "token #{API_TOKEN}", "Accept" => "text/plain" }
    ).then { |r| r.success? ? r.body : nil }
  end

  # Verify HMAC-SHA256 webhook signature
  def self.valid_signature?(payload, signature, secret)
    expected = "sha256=#{OpenSSL::HMAC.hexdigest('sha256', secret, payload)}"
    ActiveSupport::SecurityUtils.secure_compare(expected, signature)
  end
end

Webhook Handler

ruby# app/controllers/webhooks_controller.rb
class WebhooksController < ActionController::API
  skip_before_action :verify_authenticity_token

  def gitea
    secret = ENV["GITEA_WEBHOOK_SECRET"]
    sig    = request.headers["X-Gitea-Signature"]
    unless GiteaService.valid_signature?(request.raw_post, sig, secret)
      return head :unauthorized
    end

    event   = request.headers["X-Gitea-Event"]
    payload = JSON.parse(request.raw_post)

    case event
    when "push"         then handle_push(payload)
    when "pull_request" then handle_pull_request(payload)
    end
    head :ok
  end
end

CI — Jenkins

Jenkins is the Continuous Integration server. It watches Gitea via webhook, runs builds and test suites on every commit, then POSTs results back to DevTeam Hub so developers can see CI status alongside their tickets and PRs.

🔧
Jenkins
CI/CD Automation Server

Open-source automation server. Runs pipelines defined as Jenkinsfile, supports parallel stages, test reporting, artifact archiving, and hundreds of plugins.

📡
JenkinsService
Rails Integration Class

Calls the Jenkins REST API to trigger builds, fetch logs, and retrieve test results. Used by CiRun model updates and the CI dashboard.

CI Pipeline Flow

Jenkinsfile (Rails Project)

groovy// Jenkinsfile — place in repo root
pipeline {
  agent any
  environment {
    RAILS_ENV = 'test'
    DEVTEAM_URL = credentials('devteam-api-url')
    DEVTEAM_TOKEN = credentials('devteam-api-token')
  }
  stages {
    stage('Setup') {
      steps {
        sh 'bundle install --jobs 4'
        sh 'bin/rails db:create db:migrate'
      }
    }
    stage('Tests') {
      steps {
        sh 'bundle exec rspec --format RspecJunitFormatter --out reports/rspec.xml'
      }
      post {
        always {
          junit 'reports/rspec.xml'
        }
      }
    }
    stage('Notify DevTeam') {
      steps {
        script {
          def result = currentBuild.result ?: 'SUCCESS'
          sh """
            curl -X POST ${DEVTEAM_URL}/webhooks/jenkins \\
              -H 'Content-Type: application/json' \\
              -H 'X-Jenkins-Token: ${DEVTEAM_TOKEN}' \\
              -d '{"build_number":"${BUILD_NUMBER}","status":"${result}",
                   "branch":"${GIT_BRANCH}","commit":"${GIT_COMMIT}"}'
          """
        }
      }
    }
  }
}

Jenkins Callback Handler in Rails

ruby# app/controllers/webhooks_controller.rb (jenkins action)
def jenkins
  token = request.headers["X-Jenkins-Token"]
  return head :unauthorized unless token == ENV["JENKINS_WEBHOOK_TOKEN"]

  data = JSON.parse(request.raw_post)
  ci_run = CiRun.find_or_initialize_by(build_number: data["build_number"])
  ci_run.update!(
    status:     data["status"] == "SUCCESS" ? :passed : :failed,
    commit_sha: data["commit"],
    branch_name: data["branch"],
    finished_at: Time.current
  )
  head :ok
end

Deployments — Kamal

Kamal (formerly MRSK) is a deployment tool by Basecamp that packages the Rails app into a Docker image and deploys it to one or more remote servers with zero-downtime rolling restarts using Traefik as a reverse proxy.

🚀
Kamal 2
Zero-Downtime Deployer

Ships a Docker image to production servers, health-checks before switching traffic, and rolls back automatically on failure. No Kubernetes needed.

🔀
Traefik
Reverse Proxy / Load Balancer

Kamal manages a Traefik instance on the server to route traffic, handle TLS termination, and perform blue/green container swaps.

Video Calls — Jitsi Meet

Jitsi Meet is a fully open-source, self-hostable video conferencing platform. DevTeam Hub embeds Jitsi rooms directly into meeting pages via an iframe. Each meeting generates a unique room name, and developers can join with one click.

📹
Jitsi Meet
Video Conferencing

WebRTC-based video calls. Can use the public meet.jit.si server for free, or deploy your own Jitsi instance on any Linux server.

🔧
Jitsi External API
JavaScript SDK

The external_api.js library allows embedding a Jitsi conference in any webpage with full event handling — join/leave, recording, chat.

Meeting Model — jitsi_url

ruby# app/models/meeting.rb
class Meeting < ApplicationRecord
  def jitsi_url
    "https://meet.jit.si/#{jitsi_room}" if jitsi_room.present?
  end

  before_create do
    self.jitsi_room ||= "devteam-#{SecureRandom.hex(6)}"
  end
end

On-Prem S3 — MinIO

MinIO is a high-performance, S3-compatible object storage server you can run entirely on-prem. Rails Active Storage supports it out of the box as an S3 provider — no AWS account needed.

🗄️
MinIO
S3-Compatible Object Storage

Runs on Linux, Windows, or macOS. Exposes the full AWS S3 API — presigned URLs, multipart uploads, lifecycle policies, bucket policies, and a web UI on port 9001.

📎
Active Storage
Rails File Attachment Layer

Rails built-in. Attach files to any model, serve via redirect or direct URL. Switch backend (local disk → MinIO → AWS S3) by changing one config file.

Step 1 — Run MinIO

bash (Docker)# Run MinIO with Docker
docker run -d \
  --name minio \
  -p 9000:9000 \
  -p 9001:9001 \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=minioadmin123 \
  -v minio_data:/data \
  quay.io/minio/minio server /data --console-address ":9001"

Open http://localhost:9001, log in, and create a bucket named devteam-storage.

Step 2 — Gemfile

ruby# Gemfile
gem "aws-sdk-s3", require: false  # used by Active Storage S3 service

Step 3 — storage.yml

yaml# config/storage.yml
minio:
  service: S3
  access_key_id:     <%= ENV["MINIO_ACCESS_KEY"] %>
  secret_access_key: <%= ENV["MINIO_SECRET_KEY"] %>
  region:   us-east-1          # MinIO ignores region but needs a value
  bucket:   devteam-storage
  endpoint: "http://minio.local:9000"
  force_path_style: true       # required for MinIO (no virtual-hosted-style)

Using Active Storage in a Model

  has_many_attached :attachments   # multiple files

  rails_blob_url(file, disposition: "attachment") if file.attached?
end

ASP.NET Project — Dockerfile

If you have a companion ASP.NET application (e.g. a Windows service, print server, or API), here is a production-ready multi-stage Dockerfile for both ASP.NET Core Web API and a Worker Service, plus how to connect it to MinIO and the DevTeam Hub API.

ASP.NET Core Web API Dockerfile

dockerfile# ── Stage 1: Restore & Build ───────────────────────────────────────────────
FROM mcr.microsoft.com/dotnet/sdk:8.0 AS build
WORKDIR /src

# Copy csproj and restore
COPY ["MyApi/MyApi.csproj", "MyApi/"]
RUN dotnet restore "MyApi/MyApi.csproj"

# Copy all source and build
COPY . .
WORKDIR /src/MyApi
RUN dotnet build -c Release -o /app/build

# ── Stage 2: Publish ──────────────────────────────────────────────────────
FROM build AS publish
RUN dotnet publish -c Release -o /app/publish \
    /p:UseAppHost=false

# ── Stage 3: Runtime ──────────────────────────────────────────────────────
FROM mcr.microsoft.com/dotnet/aspnet:8.0 AS final
WORKDIR /app

# Non-root user
RUN adduser --disabled-password --gecos "" appuser
USER appuser

COPY --from=publish /app/publish .

EXPOSE 8080
EXPOSE 8081

HEALTHCHECK --interval=30s --timeout=10s CMD \
  curl -f http://localhost:8080/health || exit 1

ENTRYPOINT ["dotnet", "MyApi.dll"]

Connecting ASP.NET to MinIO (AWSSDK.S3)

csharp// Program.cs
using Amazon.S3;

builder.Services.AddSingleton<IAmazonS3>(_ =>
{
    var config = new AmazonS3Config
    {
        ServiceURL       = Environment.GetEnvironmentVariable("MINIO_ENDPOINT"),
        ForcePathStyle   = true,   // required for MinIO
        UseHttp          = true
    };
    return new AmazonS3Client(
        Environment.GetEnvironmentVariable("MINIO_ACCESS_KEY"),
        Environment.GetEnvironmentVariable("MINIO_SECRET_KEY"),
        config);
});

csharp// FileService.cs — upload a file to MinIO
public class FileService(IAmazonS3 s3)
{
    public async Task<string> UploadAsync(Stream stream, string key)
    {
        await s3.PutObjectAsync(new PutObjectRequest
        {
            BucketName  = "devteam-storage",
            Key         = key,
            InputStream = stream,
            ContentType = "application/octet-stream"
        });
        return $"http://minio.local:9000/devteam-storage/{key}";
    }
}

Connecting ASP.NET to DevTeam Hub API

csharp// DevTeamClient.cs — call the DevTeam Hub REST API
public class DevTeamClient(HttpClient http)
{
    public async Task UpdateTicketAsync(int ticketId, string status)
    {
        var payload = JsonSerializer.Serialize(new { ticket = new { status } });
        http.DefaultRequestHeaders.Add("X-Api-Token",
            Environment.GetEnvironmentVariable("DEVTEAM_API_TOKEN"));
        await http.PutAsync(
            $"/api/v1/tickets/{ticketId}",
            new StringContent(payload, Encoding.UTF8, "application/json"));
    }
}

// Register in Program.cs
builder.Services.AddHttpClient<DevTeamClient>(c =>
    c.BaseAddress = new Uri(Environment.GetEnvironmentVariable("DEVTEAM_URL")));
